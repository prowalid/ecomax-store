# دليل اختبار الضغط (Load Testing)

تم تصميم هذا الاختبار لمحاكاة ضغط المستخدمين على المتجر الإلكتروني. نستخدم أداة `Artillery` والتي يمكن تشغيلها مباشرة عبر `npx` دون الحاجة لتنصيبها عالميًا.

## كيفية التشغيل محليًا

تأكد أولًا من أن خادم الواجهة البرمجية (API) يعمل:
```bash
cd server
npm run dev
```

ثم افتح نافذة طرفية أخرى من جذر المشروع وقم بتشغيل الاختبار:

```bash
npm run test:load
```

لتغيير الرابط المستهدف (إذا كان الخادم يعمل على منفذ آخر):
```bash
TARGET=http://localhost:8000 npm run test:load
```

## السيناريوهات المضمنة

يحتوي ملف `ecommerce-load.yml` على مرحلتين أساسيتين:
1. **Browse Catalog**: يتصفح الأقسام والمنتجات (يمثل 70% من الزوار).
2. **Browse and Checkout**: يتصفح ثم يحاول إنشاء طلب ببيانات عميل ثابتة ومنتج ملتقط من واجهة المنتجات لضمان عمل إنشاء الطلب تحت الضغط (يمثل 30% من الزوار).

## قراءة النتائج (Artillery Report)

بعد انتهاء الاختبار، سيطبع Artillery تقريراً، أهم ما يجب مراقبته:

- **http.codes.200 / http.codes.201**: عدد الطلبات الناجحة (يجب أن يكون الأغلبية الساحقة).
- **http.codes.5xx**: عدد أخطاء الخادم (أهم مؤشر لاكتشاف المشاكل تحت الضغط، يجب حله إذا ظهر).
- **http.codes.4xx**: مهم أيضًا، لأن ظهوره بكثرة قد يعني أن سيناريو إنشاء الطلب لا يطابق تحقق الخادم أو أن البيانات الملتقطة من API غير كافية.
- **vusers.failed**: عدد المستخدمين الوهميين الذين لم يكملوا السيناريو.
- **http.response_time.p95**: زمن استجابة 95% من الطلبات بالملي ثانية (يجب أن يكون تحت 1000ms للطلبات العادية، ويفضل أقل من 500ms).

## التنبيهات (Prometheus Alerts)

أثناء تشغيل الاختبار، يمكنك فتح Grafana لمراقبة أداء الخادم وقاعدة البيانات. ستعمل القواعد المحدثة في `deploy/monitoring/alerts.yml` على إصدار تنبيهات Critical أو Warning في الحالات التالية:

- ارتفاع معدل 5xx لأكثر من 5% (Warning) أو 10% (Critical).
- ارتفاع زمن الاستجابة p95 لأعلى من 1.5 ثانية (Warning) أو 3 ثوانٍ (Critical).
- وصول الطلبات المتزامنة (Inflight requests) لرقم مرتفع جداً.

> **ملاحظة:** هذا الاختبار smoke/performance baseline أولي. الهدف منه قياس الاستقرار العام للمسارات الأهم، وليس محاكاة كل منطق checkout النهائي أو كل تنويعات بيانات السوق.
