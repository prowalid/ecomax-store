# خريطة الطريق نحو الاعتماد الإنتاجي (Production Upgrade Roadmap)

بناءً على تقييم وكيل الإشراف (Codex)، هذه الخطة التفصيلية لتحويل المنصة من "مشروع ميزات ممتازة" إلى "منتج تجاري مستقر 100%" (Enterprise Grade). سنعمل عليها نقطة بنقطة.

## المحور الأول: التفاصيل التشغيلية والاحترافية (Operational Polish)
هذا المحور يركز على توفير أدوات تسهل حياة التاجر وتقنعه بترك المنصات المدفوعة.

- [x] **1.1 ميزة التصدير (Export to CSV/Excel):**
  - إضافة زر وتفعيل دالة لتصدير **الطلبات** إلى ملفات Excel/CSV (مهم جداً لتسليم الطلبات لشركات الشحن الجزائرية).
  - إضافة زر مماثل لتصدير قوائم **المنتجات** و **الزبائن**.
- [x] **1.2 الإجراءات الجماعية (Bulk Actions):**
  - تفعيل صناديق الاختيار (Checkboxes) في جدول الطلبات.
  - إضافة القدرة على تغيير "حالة الطلب" لعدة طلبات دفعة واحدة (مثال: تأكيد 10 طلبات بضغطة واحدة).
  - إضافة القدرة على حذف عدة منتجات دفعة واحدة.
- [x] **1.3 تحسين تجربة المستخدم (UX Enhancements):**
  - إضافة رسائل تأكيد (Toasts) لكل عملية حفظ، تحديث، أو حذف. (تم التحقق والتفعيل مسبقاً)
  - إضافة مؤشرات تحميل (Spinning Loaders) داخل الأزرار أثناء إرسال البيانات (لمنع الضغط المزدوج). (مفعل مسبقاً)
  - إضافة نوافذ تأكيد (Confirmation Dialogs) تنبثق قبل أي عملية حذف حساسة (منتج، تصنيف). (مفعل مسبقاً)

## المحور الثاني: الترابط والصقل البصري المتكامل (UI Cohesion)
توحيد الهوية البصرية لجميع صفحات لوحة التحكم لتتطابق مع تصميم (Dabang) الاحترافي الذي طبقناه على الرئيسية.

- [x] **2.1 صفحة الطلبات (Orders):**
  - إزالة حواف الجداول الخشنة واستبدالها بالتصميم النظيف.
  - إضافة نظام تصفية وبحث سريع وجذاب (Filters) حسب حالة الطلب.
- [x] **2.2 صفحة المنتجات (Products):**
  - تحويل طريقة عرض المنتجات لجدول احترافي مطابق للأسلوب الجديد.
  - تحسين شكل نافذة "إضافة/تعديل منتج" (النماذج Forms).
- [x] **2.3 باقي الصفحات:**
  - تحديث واجهات (التصنيفات، الزبائن، الخصومات، والصفحات) لتستخدم نفس نوع الكروت البيضاء الناصعة والظلال الناعمة.
- [x] **2.4 متفرقات:**
  - تم تحقيق توحيد بصري كامل (UI Cohesion).

## المحور الثالث: الاستقرار والثقة (Stability & Trust)
ضمان عدم انهيار المنصة تحت الضغط وعدم فقدان البيانات.

- [x] **3.1 تسجيل الأخطاء (Logging & Monitoring):**
  - إضافة نظام تسجيل أخطاء مبسط (Winston أو Morgan) في الباك-إند لتسجيل الأعطال والتأكد من إمكانية تتبع الأخطاء بدلاً من صمت النظام.
- [x] **3.2 النسخ الاحتياطي (Automated Backups):**
  - تم إعداد سكريبت باش `backup.sh` لأخذ نسخ احتياطية مضغوطة لقاعدة البيانات وحذف النسخ القديمة وتجهيزه لـ Cron Jobs.
- [x] **3.3 اختبار التحمل (Stress Testing & Race Conditions):**
  - تم مراجعة الكود: عملية الطلب والدفع وأكواد الخصم محمية بالكامل بواسطة أقفال قاعدة البيانات `FOR UPDATE` على مستوى Postgres، مما يمنع حدوث مشاكل تزامن (Race Conditions) بشكل قطعي.
